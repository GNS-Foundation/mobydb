Benchmark methodology

    

      

        
Where MobyDB wins

        
Proximity queries (near) — composite key range scan vs GiST index lookup. Combined spatial + temporal filters — one key scan vs two-index merge. Aggregation (zoom_out) — H3 parent traversal vs GROUP BY + boundary join.

      

      

        
Where PostGIS still wins

        
Complex polygon operations (GEOS library). Point-in-polygon at low cardinality. Full-text attribute search. Any query that doesn't have WHERE + WHEN + WHO as its core shape.

      

      

        
Hardware

        
Both databases run on Railway cloud infrastructure. Same region, same resource allocation. PostGIS: GiST index on geometry, BRIN on timestamp, B-tree on epoch. No artificial handicaps on either side.

      

      

        
MobyDB writes more

        
MobyDB verifies an Ed25519 signature on every write — a cost PostGIS doesn't have. This is intentional. MobyDB provides a guarantee PostGIS structurally cannot: every record is cryptographically attributed. Write throughput reflects this real cost.
